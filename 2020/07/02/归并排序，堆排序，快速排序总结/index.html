归并排序

特性

空间复杂度 O(n)

时间复杂度 O(nlogn)

稳定性 稳定

k路归并的归并趟数 ceil(log_k n)

VisuAlgo

实际运行中往往是对左边递归递归递归递归完再进行右边的递归排序，而考试中是一次就排序一层，注意这个差别。

堆排序

特性

空间复杂度 O(1)

时间复杂度

堆排序O(nlogn)

建堆O(n)

插入,删除O(log2n)

向下调整O(h）(其实就是O(log2n)吧，二叉树诶。

稳定性 不稳定

VisuAlgo

code

向下调整

  1. 将当前节点V与其左右孩子相比较，与其中权值更大且大于V的结点交换（如果有的话）。
  2. (已经在下面的)结点V继续与孩子比较，直到结点V的孩子的权值都比结点V的权值小或者不存在。

建堆

从floor(n/2)到1：向下调整（到叶结点）

堆排序

  1. 建堆
  2. 交换堆底堆顶
  3. 堆顶向下调整（到叶）

2.3循环

快速排序

特性

空间复杂度

最好O(log2n)

最坏O(n)

平均O(log2n)

时间复杂度

平均O(nlog2n)

最差O(n^2) 基本有序或基本逆序的情况下

稳定性 不稳定

“内部排序算法中最快的。”

“每次排序后基准元素总能放到最后位置上”
