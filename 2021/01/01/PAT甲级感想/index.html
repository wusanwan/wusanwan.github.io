我喜欢犯的弱智错误总结

有参考 「为什么我的这段代码有几个测试点不通过呢」系列及常见问题FAQ

用double不要用float.
用for语句再中途break，此时会少一次++需要注意

平时处理的思路：

  1. 重读题目
  2. 看边界情况，比如只有1个
  3. 一句一句写注释看逻辑有没有问题

字符串

  1. 字符串getline如果读不进去，看看是不是忘记用getchar()吃换行符了，可能getline把上一行的给读了。

  2. ISO C++ forbids comparison between pointer and integer [-fpermiss]

    id[0]==”0”是错的，应该是id[0]=’0’

莫名其妙

  1. 没输出就凉了，scanf加&了吗?
  2. !x%2的写法是错误的，应该是!(x%2)

  3. 数组赋值错误

  4. ld.exe: cannot open output file … ： Permission denied

    打开任务管理器把进程结束了。

  5. 数值不对

    可能是控制符打错了如果一个long long的3你用%d打就会变0，也可能是==和=搞错了。

  6. vs2019 chkstk.asm报错，是开的数组太大导致的堆栈溢出，到项目→项目属性→连接器→系统→保留堆栈大小改大。

    溢出处理

防溢出的话可以用long,long long（2^64）。

1065. A+B and C (64bit) (20)-PAT甲级真题

溢出后是负的，可以判断下如果溢出了就舍弃（肯定不是题目所要的）

浮点数的精度问题到最后再截断

排序题里的score如果是int，score/1.5什么的score*1.5什么的每次运算都会截断，精度就有损失，不如先设置成double,再在排序前统一用int一次性截断，这样就避免了精度上的损失。

pow()的结果是double,转成int不能直接用int(),会截断，应该是用round()

OJ错误类型

错误

  1. 看有没有溢出，用double不用float，用long long 不用int。乘法容易溢出。
  2. 数组大小m比如给了10000，题意中有没有说你应该把数组大小放大（哈希那题放大成素数）

段错误

可以通过注释来寻找错误的语句

  1. 数组开太小了，立马后面改大到100000甚至更大（可能是题意理解错误，比如1000个通话记录可能对应2000个人。）
  2. 数组开太大了
  3. 索引超了，比如有一个2位数的字符串索引，你拿它来-‘0‘索引肯定爆炸。用sscanf或者stoi。
  4. STL的comparor在两个元素相等时必须返回false，有个专门的名词叫严格弱序（strict weak ordering）要满足它。 也就是你的cmp最后必须返回false而不是true，否则STL算法就判断出错，结果就不可知了。

超时

  1. 超时的话其实大概率是思路的问题，看一下目前思路的复杂度，能不能给它降个级。

    比如O(n^2），可以想下有没有O(nlogn),O(n)。logn的话很自然的就想到二分。

  2. 看有没有搜索，有搜索看是不是顺序，是顺序的话就加一个二分。

  3. 如果有map的话可以改成unordered_map头文件里也要改。
  4. 可能是n和m相差太大，需要对读入的数字做一次筛选再找前n个
  5. 避免重复查找，空间换时间
  6. 没查询就别先排序
  7. cout改成printf
  8. 传递参数改成引用模式bool cmp(const pair&a, const pair&b)

格式错误

  1. 有时根据题意，就算第一行为0，第二行也得输出空行，否则会有报错。
